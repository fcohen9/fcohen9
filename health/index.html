Let's Get Healthy!

A Web App for Total Health

process

The problem I wanted to solve with this product was the lack of health apps that are actually healthy. The majority of current diet apps encourage unhealthy eating habits and lifestyles, like counting calories, cutting out food groups, and equating health with weight loss.

I started this project figuring out my problem statement

Our user struggles to get healthier because they do not know how. Current health apps donâ€™t actually encourage healthy behavior, shaming our user and putting too much emphasis on weight loss. Our solution should deliver an easy to use, not calorie or weight-loss centric, and positive application to help our user learn to be healthier.

I started with some competitive analysis to confirm my assumptions about current health apps and their users.

Lose It!

Pros

  • It's easy to record food, water, and exercise
  • Shows user trends
  • Not as ugly as other weight loss apps
  • Allows users to input exercise data

Cons

  • A calorie counting app
  • It lets you set weight loss goals
  • Gives you "daily budget" of calories
  • Has under 400/500 calorie meal plans
  • Suggests number of calories per meal
  • Only uses exercise data for calorie counting

My Fitness Pal

Pros

  • Initial options include maintain weight and gain weight; Does not assume user goal is to lose weight
  • Asks you questions like "How active are you?" (More emphasis on actual health, not weight loss)
  • Easy to set up and get plan
  • Doesn't let set calorie intake go below 1200
  • Has recipes that are healthy, but not ridiculously low calorie

Cons

  • Still focused on calories
  • Stale UI
  • User must use app every day for it to be useful

Through this, my assumption that health apps are very weight-loss centric was confirmed. My assumption that people of all genders, races, and ages use health apps was also confirmed.

I made a couple of quick user personas in order to narrow down main user goals:

From these user personas, I started brainstorming. For example:

  • Quick to learn and use
  • Take the numbers out of it; simplify
  • Set custom goals
  • Some kind of log, worksheet, or other major interactive element
  • Mental health included
  • Illustrations aren't really a feature, but I wanted this app to be as positive as possible and thought that fun illustrations would help.

I decided to make a web app because I wanted the experience to be more like filling out a (fun) worksheet or playing a game.

I began with a simple userflow, and then thought about the visuals to complement my interactions.

After wireframing, I made a higher fidelity prototype based on my sketches and my list of features.

In order to create a health app without any dieting or restricting, I decided to first create an info page with information about why different aspects of health are important in many different ways. I really wanted to emphasize that maintaining a healthy relationship with food and exercise is critical.

To make this app more fun, I decided that it should be very interactive so that the user is consistently engaged. I created a landing page with a fun animation to introduce users to the feel of the app

I did end up doing a couple of illustrations, and I used calming colors.

I was lucky enough to be able to do user testing with this web app, and I got helpful feedback like that it was slightly confusing where the user was within the app. It was great to be able to do user testing because my users poked a lot of holes, but I was then able to go and fix them. For example, I changed my nav structure quite a bit from my medium fidelity prototype to my final one.
